## 4.1 RQ1: Behavioral effect 
**RQ1** tests whether the shared runtime charter and benchmark-specific harness logic materially change agent behavior and task outcomes under fixed budgets.

The first result in that process metrics move much more than resolved rate. On SWE-Bench Verified, the TRAE and Live-SWE rows stay within a narrow performance band, but Full IHR produces much larger changes in tokens, calls and runtime than either ablation.

The trajectory-level evidence shows that Full IHR is not a prompt wrapper. for TRAE, Full IHR sharply increases tool calls, LLM Calls, and runtime, and table 4 shows that about 90% of prompt tokens completion tokens, tool calls, and LLM calls occur in delegated child agents rather than in the runtime-owned parent thread.

Live-SWE is the lighter regime of the same mechanism: it raises process cost more moderately, but it still pushes the run toward a more explicit staged workflow than either ablation.

The next result is that most SWE instances do not flip. Across both TRAE and Live-SWE, more than 110 of 125 stitched SWE samples agree between Full IHR and each ablation.

## 4.2: RQ2: Harness pattern ablations
**RQ2** asks whether, once harness patterns are made explicit, they can be composed and ablated as modules under a shared substrate.

The first pattern is that module effects concentrate on a small solved frontier rather than shifting the whole benchmark uniformly. Most tasks are either solved robustly by nearly all conditions or remain unsolved across conditions, so the informative differences come from boundary cases that flip under changed control logic.

RQ2 should therefore be read as a study of how modules reshape the frontier of difficult cases, not just as a ranking over mean scores.
## RQ3: Code-to-text harness migration
RQ3 is a paired migration study: each harness appears in two realizations (source code vs. reconstructed NLAH), evaluated under a shared reporting schema.

The target is task-level equivalence—comparable exposed logic, contracts, and benchmark-facing artifacts—not identical internal traces. On OSWorld, the migrated OSSymphony realization reaches 47.2 versus 30.4 for the native code harness.
