<center><h1>Abstract</h1></center>
Agent performance increasingly depends on *harness engineering*, yet harness design is usually buried in controller code and runtime specific conventions, making it hard to transfer, compare, and study as as scientific object. they ask whether the high-level control logic of an agent harness can instead be externalized as a portable executable artifact, They introduce **Natural Language Agent Harnesses (NLAHs)**, which express harness behavior in editable natural language, and **Intelligent Harness Runtime (IHR)**, a shared runtime that executes these harnesses through explicit contracts, durable artifacts, and lightweight adapters. Across coding and computer-use benchmarks, they conduct controlled evaluations of operational viability, module ablation, and code-to-text harness migration.
