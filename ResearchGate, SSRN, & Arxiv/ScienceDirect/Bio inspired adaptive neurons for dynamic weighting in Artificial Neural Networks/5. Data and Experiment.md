# 5. Data and Experiment

Bab ini memaparkan metodologi eksperimen, karakteristik dataset yang digunakan untuk pengujian, serta desain skema pemangkasan (*pruning*) parameter untuk mengoptimasi model berorde tinggi. Eksperimen ini dirancang untuk melakukan perbandingan yang adil antara model yang diusulkan dengan model Multi-Layer Perceptron (MLP) konvensional.

---

## 5.1. Dataset Evaluasi (Penn Machine Learning Benchmarks)

Evaluasi kinerja model dilakukan menggunakan **145 dataset klasifikasi** yang bersumber dari Penn Machine Learning Benchmarks (PMLB) oleh [[Olson dkk. (2017)]]. Kumpulan dataset ini dipilih karena memiliki keragaman yang sangat tinggi dalam berbagai aspek statistik, menjadikannya standar pengujian yang ketat untuk mengukur keandalan dan generalisasi model.

Karakteristik statistik dari 145 dataset tersebut meliputi:
*   **Ukuran dataset (Jumlah sampel):** Berkisar dari dataset sangat kecil (42 sampel) hingga dataset skala besar (105.908 sampel).
*   **Dimensi fitur:** Memiliki jumlah fitur masukan dari yang sangat rendah (2 fitur) hingga dimensi tinggi (1.000 fitur).
*   **Kompleksitas label:** Mencakup tugas klasifikasi biner (2 kelas) hingga klasifikasi multi-kelas (hingga 26 kelas).

Keragaman ini (seperti divisualisasikan pada Gambar 2 dalam jurnal) memastikan bahwa model adaptif yang diuji tidak hanya unggul pada tipe data tertentu, melainkan tangguh di berbagai kondisi data masukan yang berbeda. Detail spesifik mengenai nama-nama dataset komparatif disajikan pada [[9. Information]].

## 5.2. Kerangka Kerja Eksperimen (Experiment Framework)

Prosedur pengujian dirancang secara ketat untuk meminimalkan bias varians hasil pelatihan. Langkah-langkah eksperimen untuk setiap dataset adalah sebagai berikut:

1.  **Pemisahan Data:** Dataset dibagi menjadi data pelatihan (*training set*) dan data pengujian (*test set*).
2.  **Pelatihan MLP Baseline:** Model MLP standar dilatih sebanyak **10 kali** dengan inisialisasi acak yang berbeda. Akurasi pengujian terbaik dari 10 kali pengujian tersebut dicatat sebagai performa baseline.
3.  **Pelatihan Model Chebyshev:** Model adaptif Chebyshev dilatih sebanyak **10 kali** pada pembagian data (*split*) yang sama persis dengan MLP. Akurasi pengujian terbaik dicatat sebagai hasil model yang diusulkan.

Mekanisme pelatihan berulang (10 kali evaluasi) ini diilustrasikan pada diagram alur Gambar 3 dalam jurnal untuk memastikan bahwa perbandingan tidak terpengaruh oleh faktor keberuntungan inisialisasi acak (*lucky seed*).

### 5.2.1. Arsitektur Model
Untuk menjaga keadilan pengujian, kedua model dikonfigurasi dengan topologi tiga lapisan yang identik:
*   **Lapisan Tersembunyi 1 (Hidden Layer 1):** Terdiri dari 4 neuron.
*   **Lapisan Tersembunyi 2 (Hidden Layer 2):** Terdiri dari 2 neuron.
*   **Lapisan Keluaran (Output Layer):** Jumlah neuron disesuaikan dengan jumlah kelas pada dataset yang diuji.

Pada model adaptif Chebyshev, transformasi ekspansi polinomial berderajat $k = 3$ diterapkan pada masukan di setiap lapisan, sedangkan MLP baseline menggunakan kombinasi linier statis standar.

### 5.2.2. Pengaturan Pelatihan (Training Setup)
Kedua jaringan saraf dilatih menggunakan konfigurasi hiperparameter yang seragam:
*   **Laju Pembelajaran (Learning Rate):** $0.001$
*   **Fungsi Aktivasi:** ReLU (Rectified Linear Unit)
*   **Pengoptimal:** Adam
*   **Jumlah Epoch:** 500

## 5.3. Strategi Pemangkasan Parameter (Pruning)

Penggunaan orde polinomial $k$ yang tinggi (misalnya $k = 6$) memperkaya kapasitas representasi model tetapi meningkatkan risiko kelebihan parameter (*overparameterization*) dan overfitting. Untuk mengatasinya, penulis memperkenalkan dua strategi pemangkasan setelah pelatihan (*post-training pruning*):

### 5.3.1. Strategi 1: Pemangkasan Nilai Ambang Parameter (Parameter Thresholding)
Strategi pertama adalah melakukan pemangkasan berbasis elemen di mana koefisien yang memiliki magnitudo di bawah nilai ambang batas tertentu akan dibuang (dijadikan nol). Pemangkasan dilakukan secara bertahap lapis demi lapis (*layer-wise*):

$$W^{(l*)} = \text{Prune}(W^{(l)} \mid W^{(1*)}, \dots, W^{(l-1*)}) \tag{1}$$

Di mana $W^{(l*)}$ merepresentasikan matriks bobot yang telah dipangkas pada lapisan $l$. Parameter yang telah dipangkas akan dibekukan selama proses penyesuaian halus (*fine-tuning*), sementara parameter non-zero tetap dapat diperbarui untuk memulihkan akurasi model.

### 5.3.2. Strategi 2: Pemangkasan Kelompok Berbasis Struktur Chebyshev (Chebyshev-aware Grouped Pruning)
Strategi kedua memanfaatkan struktur matematis dari ekspansi polinomial dengan memperlakukan semua koefisien basis untuk satu fitur masukan sebagai satu kelompok. Penulis menghitung norma L2 dari seluruh koefisien $\{c_{i,j}\}$ untuk input ke-$i$:

$$\|w_i\|_2 = \sqrt{\sum_{j=0}^{k} c_{i,j}^2} \tag{2}$$

Jika nilai norma kelompok $\|w_i\|_2$ berada di bawah nilai ambang batas pemangkasan, maka seluruh rangkaian koefisien $\{c_{i,j}\}$ untuk fitur tersebut akan dibuang bersama-sama. Strategi pemangkasan kelompok ini sangat efektif karena mampu mengeliminasi kontribusi fitur masukan yang tidak penting secara keseluruhan, menjaga interpretabilitas model, serta meningkatkan efisiensi komputasi pada lapisan masukan.

## 5.4. Analisis Hubungan Input-Bobot dan Batas Keputusan

### 5.4.1. Perilaku Adaptasi Bobot
Hasil eksperimen menunjukkan bahwa bobot adaptif model Chebyshev berubah secara mulus (*smooth*) seiring dengan variasi nilai masukan. Sebagai contoh, Gambar 4 dalam jurnal menampilkan visualisasi perubahan nilai bobot pada neuron lapisan pertama untuk dataset `phoneme`. Sifat kontinuitas ini merupakan bukti nyata dari pengaruh positif sifat stabil Polinomial Chebyshev.

### 5.4.2. Visualisasi Batas Keputusan (Decision Boundaries)
Keunggulan kapasitas representasi non-linear model adaptif diilustrasikan dengan jelas melalui visualisasi batas keputusan pada dataset yang menantang:
*   **Dataset `solar_flare_1`:** Memiliki 5 kelas non-linear. Seperti diperlihatkan pada Gambar 5 dalam jurnal, batas keputusan yang dibentuk oleh model Chebyshev jauh lebih presisi dan menghasilkan area salah klasifikasi (berwarna pink) yang jauh lebih sedikit dibandingkan MLP baseline.
*   **Dataset `Hill Valley without noise`:** Seperti diperlihatkan pada Gambar 6 dalam jurnal, model Chebyshev berhasil menangkap pola gelombang lembah-bukit yang rumit, sedangkan MLP konvensional gagal memisahkan kelas-kelas tersebut dengan baik karena keterbatasan kapasitas pemisahan linier lokalnya.
