# 6. Security and privacy in Agentic AI systems

Bagian ini memperdalam aspek teknis [[Application Security]] dan [[Model Privacy]] untuk [[Agentic AI]] dan [[AMAS]]. AMAS memiliki attack surface lebih besar daripada pipeline LLM tunggal karena mencakup data antargens, tool calls, memory store, orchestrator, prompt, hasil retrieval, log, dan keputusan runtime.

Tabel 8 membandingkan benchmark seperti [[HarmBench]], [[ToolBench]], [[AgentBench]], [[GAIA]], [[WebArena]], [[HELM]], dan [[MLCommons AI Safety]]. Polanya menunjukkan bahwa tidak ada benchmark tunggal yang mencakup semua risiko AMAS.

> [!important] Temuan Utama
> Security dan privacy dalam AMAS harus berupa defense in depth: enkripsi, access control, adversarial hardening, runtime monitoring, data minimization, differential privacy, secure computation, dan benchmark keamanan.

## 6.1 Security mechanisms

Jurnal membahas [[Encryption]], [[Access Control]], [[Adversarial Learning]], dan [[Runtime Monitoring]]. Encryption melindungi data saat berpindah atau disimpan, access control membatasi hak agen, adversarial learning meningkatkan ketahanan, dan runtime monitoring mendeteksi perilaku menyimpang setelah sistem berjalan.

[[Encryption]] menjaga data yang dipertukarkan antar agen, terutama data kesehatan, keuangan, atau data regulatif. Implementasi seperti SSL TLS, homomorphic encryption, dan secure enclaves membantu menjaga confidentiality pada message passing protocols.

[[Access Control]] penting karena AMAS terdiri dari agen dengan peran berbeda. [[Role Based Access Control]] dan [[Attribute Based Access Control]] membatasi akses berdasarkan peran, atribut, dan tingkat kepercayaan kontekstual.

> [!info] Definisi / Latar Belakang
> [[Least Privilege]] membatasi setiap agen hanya pada hak akses minimum yang diperlukan untuk tugasnya.

[[Adversarial Learning]] merespons prompt injection, poisoned tool outputs, dan malformed intermediate results. [[Runtime Monitoring]] memakai log based auditing, anomaly detection, dan trust scoring untuk mendeteksi perilaku abnormal.

## 6.2 Privacy-preserving techniques

Privasi AMAS membutuhkan [[Differential Privacy]], [[Data Minimization]], dan [[Secure Computation]]. DP melindungi individu dengan menambahkan noise terkontrol dan memakai privacy budget $\epsilon$. Data minimization membatasi cakupan, granularitas, dan durasi data yang dikumpulkan atau disimpan.

[[Secure Multi Party Computation]], [[Homomorphic Encryption]], dan [[Trusted Execution Environments]] memungkinkan komputasi atas data terenkripsi atau tersamarkan tanpa membuka data mentah.

> [!info] Definisi / Latar Belakang
> [[Privacy by Design]] berarti privasi dibangun sejak awal ke dalam arsitektur sistem, termasuk consent layer, pengaturan privasi, memory redaction, dan data minimization.

> [!warning] Limitasi / Anomali Data
> Teknik seperti homomorphic encryption dan secure computation memiliki biaya komputasi atau trade off performa, sehingga perlu dipilih sesuai sensitivitas domain.

## 6.3 Evaluation benchmarks and metrics for Agentic AI security

Benchmark keamanan mencakup [[HarmBench]] dan [[JailbreakBench]] untuk prompt injection dan jailbreak, [[ToolBench]] dan [[API Bank]] untuk tool misuse, serta [[AgentBench]] dan [[GAIA]] untuk skenario multi agen.

Metrik utama adalah [[Attack Success Rate]], [[Robustness Score]], [[Defense Effectiveness Rate]], [[False Positive Rate]], dan [[False Negative Rate]]. ASR mengukur persentase serangan berhasil; robustness mengukur ketahanan terhadap kondisi adversarial; defense effectiveness mengukur serangan yang terdeteksi dan dimitigasi.

> [!question] Bagian Tidak Jelas
> Jurnal menggunakan singkatan CSS untuk [[Component Synergy Score]] dan juga menyebut Composite Security Score. Interpretasi terbaiknya: Component Synergy Score adalah metrik formal, sedangkan composite security score adalah gagasan agregasi keamanan.
