# MOC: Pseudo-Random Number Generation in an Agent-based Game Environment using Two-dimensional Linear Cellular Automata

> [!abstract] Apa Inti Jurnal Ini? (ELI5)
> Bayangkan sebuah permainan papan yang mirip dengan "Game of Life" milik Conway. Di papan ini, setiap kotak bisa hidup atau mati, dan aturan tertentu menentukan kotak mana yang hidup di langkah berikutnya. Sekarang, bayangkan kita memasukkan dua pemain cerdas ke dalam permainan ini — satu pemain ingin membuat sebanyak mungkin kotak hidup, dan pemain lainnya ingin membuat sebanyak mungkin kotak mati. Kedua pemain ini saling berkompetisi, dan hasilnya sangat tidak terduga karena tidak ada yang bisa mengontrol sepenuhnya.
>
> Jurnal ini memanfaatkan ketidakpastian dari kompetisi tersebut untuk membuat angka-angka acak. Seperti melempar dadu yang sangat adil — hasilnya sulit ditebak. Para peneliti menguji angka-angka acak yang dihasilkan dengan 188 tes statistik standar dari NIST, dan hasilnya menunjukkan bahwa sistem ini mampu menghasilkan bilangan acak semu berkualitas tinggi — bahkan setara atau lebih baik dari generator bilangan acak terkenal seperti Mersenne Twister dan Blum-Blum-Shub.
>
> Yang unik dari pendekatan ini adalah "keacakan" tidak hanya datang dari aturan matematika saja, tetapi juga dari persaingan antar agen cerdas yang menggunakan algoritma pencarian pohon Monte Carlo (MCTS). Semakin banyak agen yang berkompetisi, semakin bagus kualitas bilangan acak yang dihasilkan.

## Ringkasan Teknis Cepat

| Aspek | Detail |
|-------|--------|
| **Tipe Kontribusi** | Framework baru: kombinasi 2D Linear Cellular Automata (LCA) dengan lingkungan permainan berbasis agen (gamified agent-based) |
| **Area CS** | Algoritma, Cellular Automata, Game Theory, MCTS, Pseudo-Random Number Generation |
| **Masalah yang Diselesaikan** | Meningkatkan kualitas PRNG berbasis Cellular Automata melalui interaksi agen cerdas yang berkompetisi |
| **Metode Utama** | 2D Linear CA + MCTS agents (maximizer/minimizer) + Negamax backup policy + state heuristics |
| **Kompleksitas / Cost** | Trade-off signifikan: kualitas tinggi namun runtime dan memori lebih besar dari PRNG konvensional |
| **Dataset / Benchmark** | 210 file konfigurasi (125 juta bit per file), total ~25.6 GB |
| **Metrik Evaluasi** | Total Passed Tests (TPT) dari NIST Statistical Test Suite — 188 tes, skor maksimum 188 |
| **Hasil Utama** | Skor TPT hingga 188/188 (sempurna) pada mayoritas konfigurasi; kompetitif terhadap Mersenne Twister (187) dan Blum-Blum-Shub (186) |
| **Kode Tersedia?** | Tidak disebutkan secara eksplisit dalam jurnal |
| **Venue** | Computing and Informatics (DOAJ indexed) |
